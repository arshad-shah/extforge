# Support

Need help with ExtForge? Pick the channel that matches your question.

## Documentation

The full documentation site is at <https://extforge.arshadshah.com>:

- **Getting started** — installation, project scaffolding, first build.
- **Guides** — HMR, storage, messaging, CSUI, env loading, packaging.
- **Reference** — auto-generated docs for config, errors, plugins, and
  the full TypeScript API for every public entry point.

## Questions and discussion

For "how do I", "is this possible", and architectural questions, open a
**GitHub Discussion** at
<https://github.com/arshad-shah/extforge/discussions>.

Discussions are searchable, so future users with the same question
benefit from your conversation.

## Bug reports

If something is genuinely broken, file a **bug report** at
<https://github.com/arshad-shah/extforge/issues/new?template=bug_report.yml>.
Please include the smallest reproducible example you can: an
`extforge.config.ts`, the source files involved, and the exact command
that fails.

## Feature requests

Use the **feature request** template at
<https://github.com/arshad-shah/extforge/issues/new?template=feature_request.yml>.

Be specific about the use case. "I want X because Y" lands better than
"please add X".

## Security

Do **not** report security issues in public issues or discussions. See
[SECURITY.md](./SECURITY.md) for the private disclosure process.

## Commercial support

None offered at this time. ExtForge is MIT-licensed and maintained on a
best-effort basis.
